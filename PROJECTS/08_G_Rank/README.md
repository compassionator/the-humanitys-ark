# G-Rank

## Role
Seed / Product

## Goal
Build a competence reputation and learning-rank system based on real growth, proven output, and visible consequence rather than popularity, dead certificates, or institutional prestige.

## Why it matters
G-Rank exists to move beyond fake ratings and fake grades.

Builders should be ranked by builders.
Poets by poets.
Teachers by learners.
Guides by consequence.
Students by effort, growth, and real output.

It should reward:
- effort and improvement, not only final result
- practical application, not memorized performance
- community and peer recognition, not only top-down approval
- flexible path switching, not punishment for changing direction

## Current next step
Define one narrow G-Rank scoring model first, then test it in one real domain.

Recommended first domains:
- software code quality
- mentor / teacher ranking
- learner growth inside Honarestan

## Status
Draft

## Main uses
- expert / mentor ranking
- student growth ranking
- teacher and course visibility
- competence over credentials
- reputation based on lived consequence

## Design rule
Do not collapse everything into one universal score.
Ranking should remain domain-aware and context-aware.

## Relationship to other projects
- **Dorr** provides the color and consequence logic
- **Honarestan** can use G-Rank instead of dead grading
- **The Cornea** may later display G-Rank layers in AR
- broader communities may use it to evaluate real guides and real work
