# ARK Lens peer-alpha feedback template

Copy the section below for each problem or suggestion. One issue per report is easiest to review.

```text
ARK Lens build: v2026.06.019
Date:

What were you trying to do?

What did you expect?

What happened instead?

Impact: Blocked / Difficult / Minor / Suggestion
Area: Installation / Lens setup / LinkedIn / SEEK / Session / Report / Feedback / Fix Capture / Export

Steps to reproduce:
1.
2.
3.

Did it happen again? Always / Sometimes / Once

Attachments:
- Alpha Test Summary:
- Screenshot:
- Help File, only if capture failed:

Anything else that would help?
```

## Safe attachments

The Alpha Test Summary is intended for normal issue reports because it contains aggregate counts and versions only. A screenshot is useful when the visible wording or layout is the problem; crop or cover personal information first. The Fix Capture Help File is relevant only when ARK Lens cannot read a supported page, and its preview must be reviewed before sharing.

Do not attach raw page HTML, browser profile files, Chrome local-storage files, full JSON/CSV report exports, resumes, job notes, or unrelated personal data. The extension does not need those files for ordinary alpha feedback.

## Good feedback example

“On SEEK, I started a session and moved through three jobs in the same results tab. The second job did not update Last saved. Check Capture reported that the description was missing. It happened twice. Impact: Difficult. I attached the privacy-safe Alpha Test Summary, a cropped screenshot, and the reviewed Help File.”

Specific observations are more useful than a proposed code solution. If ranking feels wrong, include the relevance choice and reason you selected; do not send the full job description unless you have permission and deliberately choose to share it.
