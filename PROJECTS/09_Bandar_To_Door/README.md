# Bandar To Door

## Role
Tool

## Goal
A national logistics spine automating the flow of atoms from port to home.

## Why it matters
It is the transport and operational backbone for future-scale logistics: a national flow spine moving atoms from port to warehouse to site to home.

## Current next step
Translate lessons from private EziLoads experience into a clean public interface and specification for one port-operator golden path. Implementation may begin only after that public contract exists.

## Status
Draft

## External reference boundary

EziLoads is private external experience and reference material. Bandar To Door must not import, copy, or depend directly on private EziLoads source. Any implementation in this public repository requires an independently documented public interface, data contract, and testable boundary.
