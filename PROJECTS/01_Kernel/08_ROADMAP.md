## PHASE C — Integration (Test Rig)
Goal: Clean room where Fuel meets Noise.

- Setup: Python notebook / dedicated thread / controlled runner
- Run: Feed Noise into Kernel; force it to cite Fuel
- Audit:
  - cites correct?
  - refuses unsupported claims?
  - labels [INFERENCE] properly?
  - reduces harm + reduces noise?
- Output: pass/fail logs per scenario + per node

## PHASE D — Courier (Automation)
Goal: remove “Fly as Clipboard” bottleneck.

- Build COURIER-ONE (Python or equivalent)
- Input: paste text once → Kernel routes + cites Fuel → output structured response
- Fuel lives in folder/repo; courier reads it
